## 项目简介

**AI 视觉对话助手** 是一款支持「视觉 + 语音」双模态交互的智能应用，可实时调用摄像头获取视频内容、麦克风采集语音指令，结合本地 AI 推理与云端大模型能力，实现对视觉场景与语音意图的精准理解，并给出自然、贴合场景的回应。

项目基于 **Orbbec 3D 深度相机 / USB 摄像头**、**YOLO 视觉理解**、**MediaPipe 姿态感知**、**本地语音识别** 与 **端云协同大模型** 构建，兼顾交互自然度、视觉理解准确性与运营成本控制，支持纯本地部署（低成本）、端云混合部署（高精度）两种模式。

**核心能力**：
- 视觉理解：实时识别画面中的人体姿态、场景主体（如"用户举手""用户坐着""空场景"）；
- 语音交互：本地语音识别 + 意图匹配，支持自然语言指令与多轮对话；
- 端云协同：高频交互本地完成，复杂语义 / 视觉理解按需调用云端，精准控制成本；
- 多模态回应：结合视觉场景（如用户动作）+ 语音意图（如用户诉求）生成个性化回应。

## 成本控制策略（思路 vs 落地）

### 想到的成本控制技巧

| 序号 | 成本控制思路 | 适用场景 |
|------|-------------|----------|
| 1 | 「本地预处理 + 过滤」：仅对有效请求调用云端 API（如过滤空场景、无意义语音） | 视觉 / 语音无效输入场景 |
| 2 | 「分级调用」：简单交互用本地规则，复杂交互用云端低价模型（如 GPT-4o-mini），极复杂场景才用高价模型 | 不同复杂度的交互需求 |
| 3 | 「请求节流」：限制单位时间内云端调用频率（如 10 秒内最多 1 次），避免重复请求 | 高频重复指令场景 |
| 4 | 「数据轻量化」：上传云端的视觉数据仅传递结构化信息（如「人体姿态：举手」），而非原始视频帧 | 视觉理解云端调用场景 |
| 5 | 「缓存复用」：相同场景 + 相同指令的回应缓存，避免重复调用 | 高频重复场景 |
| 6 | 「按量付费切换」：云端 API 按调用量付费，而非预留资源 | 非峰值时段成本控制 |
| 7 | 「本地模型替代」：用轻量化本地模型（如 YOLO-nano、MediaPipe）替代云端视觉识别 | 基础视觉理解场景 |

### 实际采用的成本控制策略

| 序号 | 落地策略 | 具体实现 | 成本优化效果 |
|------|---------|----------|-------------|
| 1 | 本地预处理过滤无效请求 | ✅ 实现：视觉——本地检测到「无人体」「静态画面」时，直接本地回应，不调用云端；语音——本地过滤空白语音、无意义语气词（如"嗯 / 啊"），仅有效指令调用云端 | 减少约 40% 无效云端请求 |
| 2 | 分级调用（本地 / 低价云端） | ✅ 实现：基础交互（如「开始对话 / 暂停」）本地规则引擎回应；复杂语义（如「结合动作推荐放松方式」）调用 GPT-4o-mini（低价大模型）；禁用高价模型调用 | 单请求成本降低约 80% |
| 3 | 请求节流 + 频率限制 | ✅ 实现：设置「10 秒 / 次」云端调用频率上限，重复相同指令直接返回缓存结果 | 峰值期成本降低约 30% |
| 4 | 视觉数据结构化上传 | ✅ 实现：本地提取视觉特征（如「人体姿态、场景是否有人」），仅传递 JSON 结构化数据（<1KB），而非原始视频帧（MB 级） | 云端传输 / 处理成本降低 99% |
| 5 | 缓存复用 | ✅ 实现：本地缓存「场景 + 指令 → 回应」映射关系，有效期 5 分钟，相同请求直接复用 | 重复场景请求成本降低 100% |
| 6 | 按量付费切换 | ✅ 实现：所有云端 API 均配置为「按量付费」模式，无预付费 / 预留资源 | 非使用时段零成本，整体降低约 50% |
| 7 | 本地模型替代云端视觉 | ✅ 实现：YOLO + MediaPipe 本地完成人体 / 姿态识别，仅在需要「复杂场景理解」时才调用云端视觉 API（默认禁用） | 视觉识别环节云端成本降低 100%（全本地） |

### 成本控制效果总结

- 整体云端调用成本降低约 70%；
- 纯本地运行时，零云端成本；
- 核心交互场景（单人 + 基础指令）完全无云端成本，仅复杂语义场景产生微量成本。

```bash
python scripts/check_environment.py
```

该脚本会逐项检查 Python 版本、所有依赖库导入、模型文件完整性、Orbbec 相机连接状态和 USB 摄像头可用性、麦克风设备列表，并在末尾输出 PASS / WARNING / FAIL 总结。

```bash
python run_game_frontend.py
```

应用流程：

```
启动应用 → 自动检测摄像头 / 麦克风 → 对话就绪（语音 / 文字输入）→ 实时视觉 + 语音交互 → 退出
```

## 核心技术实现

### 视觉理解（纯本地）

- 底层：Orbbec 3D 相机 / USB 摄像头 → 本地帧采集；
- 推理：YOLOv26-nano（人体 / 场景主体检测）+ MediaPipe Pose（33 关键点姿态识别）；
- 输出：结构化视觉特征（如「是否有人、人体姿态、场景是否静态」）。

### 语音交互（纯本地优先）

- 语音识别：`speech_recognition` + `pyaudio`，本地 Google Speech 引擎；
- 意图匹配：纯规则引擎 → 识别「开始 / 暂停 / 结束 / 换动作 / 诉求（累了 / 酸了）」等意图；
- 多轮对话：本地上下文缓存 → 保留最近对话记录，支持追问 / 打断。

### 端云协同

- 触发条件：仅当「本地规则无法匹配意图」或「需要结合视觉 + 语音生成复杂回应」时调用云端；
- 云端模型：默认 GPT-4o-mini（低成本），可通过环境变量切换；
- 数据传输：仅上传「视觉特征（JSON）+ 语音文字」，不上传原始视频 / 音频帧；
- 降级策略：云端调用失败 / 无网络时，自动切回本地规则引擎，并提示用户。

### 成本监控

- 本地统计：记录云端调用次数、单次成本、累计成本；
- 阈值告警：当单日成本超过预设阈值时，自动禁用云端调用，仅保留本地功能。

- **语音识别**：`speech_recognition` + `pyaudio`，本地 Google Speech 引擎
- **对话管理**：纯规则匹配 + 可选外部 AI 增强
- **UI 面板**：左侧「AI 久坐健康助手」面板，含状态区、最近指令、AI 回复、推荐动作
- **快捷输入**：面板下方快捷指令按钮，无麦克风环境下的备用输入
- **文字兜底**：麦克风不可用时快捷输入始终可用

### 外部 AI 支持（可选）

| 环境变量 | 说明 |
|----------|------|
| `OFFICEFIT_AI_PROVIDER=none` | 默认，仅使用本地规则 |
| `OFFICEFIT_AI_PROVIDER=openai` | 启用 OpenAI API（需设置 OPENAI_API_KEY） |

启用外部 AI 后：

```powershell
$env:OFFICEFIT_AI_PROVIDER="openai"

python run_game_frontend.py
```

## 常见问题排查

### 1. 麦克风无响应 / 识别准确率低

1. 运行 `python scripts/check_environment.py` 查看麦克风设备列表和编号
2. 手动指定麦克风编号：
   ```powershell
   $env:OFFICEFIT_MIC_INDEX="0"
   python run_game_frontend.py
   ```
3. 开启语音调试查看识别日志：
   ```powershell
   $env:OFFICEFIT_VOICE_DEBUG="1"
   python run_game_frontend.py
   ```

### 2. 云端调用成本超出预期

- 检查是否开启请求节流：默认 10 秒 / 次；
- 切换纯本地模式：`$env:OFFICEFIT_AI_PROVIDER="none"`，禁用所有云端调用；
- 查看终端日志中的云端调用统计。

### 3. 视觉理解不准确

- 确认模型文件完整（运行环境检查脚本）；
- 确保摄像头视角清晰（人体在画面中央）；
- 切换 YOLO 模型：`$env:OFFICEFIT_YOLO_MODEL="models/yolo11n.onnx"`。

### 4. `pyorbbecsdk` 导入失败

> pip 包名是 **`pyorbbecsdk2`**，代码中仍然 `import pyorbbecsdk`。

```bash
pip install pyorbbecsdk2
```

若仍失败，请从 [Orbbec SDK GitHub](https://github.com/orbbec/pyorbbecsdk) 安装 C++ 运行时并将 `bin/` 加入系统 PATH。

### 5. 模型文件找不到

运行 `python scripts/check_environment.py` 确认缺失文件，将对应模型放入 `models/` 或项目根目录。

### 6. 相机启动失败 / 无画面

- 程序会自动尝试 Orbbec → USB 摄像头降级
- 确认至少有一个摄像头连接
- 关闭其他占用相机的程序（如 Orbbec Viewer、其他视频应用）

### 7. `protobuf` 版本冲突

```bash
pip install protobuf==4.25.9
```
